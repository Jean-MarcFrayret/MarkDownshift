# Contributing to MarkDownshift

Thank you for helping improve MarkDownshift.

## Reporting a bug or suggesting a feature

Open a GitHub issue and include your macOS and MarkDownshift versions, clear reproduction steps, expected and actual behavior, and a screenshot when useful. Please remove private document content from examples.

## Making a code contribution

1. Fork the repository and create a branch from `main`.
2. Keep the change focused and avoid unrelated formatting changes.
3. Build with `./scripts/build-app.sh`.
4. Test relevant document workflows, including Finder opening and multiple windows.
5. Open a pull request explaining the problem and solution.

Development requires macOS 14 or newer and Command Line Tools for Xcode. By contributing, you agree that your contribution is provided under the MIT License.
